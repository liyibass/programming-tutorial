# [rust-1-2] 基本型別：整數、浮點數、布林、字元，與「為什麼型別這麼講究」

> **本章目標**：認識 Rust 的基本純量型別，並理解為什麼 Rust 對「一個數字到底佔幾個 bit、有沒有正負號」這麼斤斤計較。

## 你會學到

- Rust 的四種純量型別：整數、浮點數、布林、字元
- 整數為什麼分這麼多種（`i32`、`u8`、`u64`…）
- Rust 怎麼大多時候自己推斷型別，何時要你明講
- `char` 為什麼能放 emoji

## 概念說明

### 為什麼 Rust 對型別這麼講究？

在 TypeScript 你寫 `let n = 5`，它就是個 `number`，不太需要管它佔多少記憶體。但 Rust 是**系統語言**——它要讓你能精準掌控「這個數字在記憶體裡到底佔幾個位元組」，因為這直接關係到效能與正確性。

一個比喻：訂便當時，「大碗」和「小碗」用的飯量不同、價錢不同。Rust 的整數型別就像不同碗的大小——你（或編譯器）要決定「這個數字用多大的碗裝」：

```
i8   小碗，能裝 -128 ~ 127
i32  中碗（最常用的預設），約 ±21 億
u8   小碗但只裝正數，0 ~ 255
u64  大碗只裝正數，超大
```

`i` 代表 **i**nteger 且**有正負號**（signed），`u` 代表**無正負號**（**u**nsigned，只有 0 和正數）；後面的數字是**佔幾個 bit**。

## 程式碼範例

### 整數

```rust
fn main() {
    let age: u32 = 30;        // 無號 32 位元，年齡不會是負的
    let temperature: i32 = -5; // 有號，可以是負的
    let big: u64 = 10_000_000; // 底線只是分隔，方便讀，等於 10000000
    println!("{} {} {}", age, temperature, big);
}
```

說明：`: u32`、`: i32` 這種寫在變數後的就是**型別註解**。如果你不寫，Rust **整數預設用 `i32`**（中碗，速度與範圍都實用）。底線 `_` 可以插在數字裡當千分位分隔，只是給人看的，不影響值。

### 浮點數（有小數的）

```rust
fn main() {
    let pi = 3.14159;          // 預設是 f64（雙精度）
    let ratio: f32 = 0.5;      // 也可以指定 f32（單精度，較省空間）
    println!("{} {}", pi, ratio);
}
```

有小數點的數字是浮點數，**預設 `f64`**。提醒一個跨語言的共同陷阱：浮點數有精度限制，`0.1 + 0.2` 不會剛好等於 `0.3`——這不是 Rust 的 bug，是浮點數的本質。

### 布林與字元

```rust
fn main() {
    let is_active: bool = true;       // 只有 true / false
    let letter: char = 'A';           // 單引號，單一字元
    let emoji: char = '🦀';           // char 是 4 bytes，放得下 emoji！
    println!("{} {} {}", is_active, letter, emoji);
}
```

兩個重點：

- `bool` 只有 `true` / `false`，常用在條件判斷（下一章你會大量用到）。
- `char` 用**單引號**，而且 Rust 的 `char` 佔 4 個位元組，代表一個 Unicode 字元——所以中文、甚至 emoji（像 Rust 吉祥物螃蟹 🦀）都能放進一個 `char`。

> 注意：`char`（單引號、單一字元）和「字串」（雙引號、一串字）是不同的東西。字串 [rust-6-2] 會專門講，那是 Rust 新手最常困惑的地方之一。

### 型別推斷：大多時候不用自己寫

你會發現上面有些有寫 `: 型別`、有些沒寫。Rust 很聰明，**多數時候能自己推斷型別**：

```rust
let x = 5;        // 編譯器推斷成 i32
let y = 3.0;      // 推斷成 f64
let ok = true;    // 推斷成 bool
```

所以**「非必要不寫型別註解」**是 Rust 的慣例——讓編譯器推斷，程式更乾淨。只有在編譯器無法推斷、或你想刻意指定某個型別時才寫。

## 小練習

1. 宣告一個 `u8` 變數設成 `255`，印出來。接著試著設成 `256`，看編譯器怎麼說（`u8` 最大就是 255）。
2. 算算看：宣告兩個浮點數 `0.1` 和 `0.2`，相加印出，看看是不是剛好 `0.3`。
3. 宣告一個 `char` 放你喜歡的一個 emoji，印出來。再試著用單引號放兩個字元（像 `'AB'`），觀察編譯器報錯——體會「`char` 只能是單一字元」。
