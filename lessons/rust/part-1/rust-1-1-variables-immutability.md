# [rust-1-1] 變數與不可變性：為什麼 Rust 預設變數「不能改」

> **本章目標**：理解 Rust 一個和多數語言相反的設計——變數**預設不可變**，以及為什麼這個選擇能讓程式更安全。

## 你會學到

- 用 `let` 宣告變數，以及它「預設不可變」是什麼意思
- 為什麼「不可變」往往是好事
- 需要可變時，怎麼用 `mut`
- 常數 `const` 與「遮蔽（shadowing）」是什麼

## 概念說明

### 預設不可變：一個刻意的選擇

在很多語言裡，變數天生就能隨便改。Rust 反過來：**用 `let` 宣告的變數，預設「不能再被改變」。**

用一個比喻：把變數想成「貼了標籤的箱子」。在 Rust，預設情況下箱子一旦裝了東西、貼上標籤，**裡面的東西就不准換**。想換內容，你必須一開始就特別聲明「這個箱子我要能換內容」。

```
預設：     箱子 x 裝了 5  →  之後想塞 6 進去 → ❌ 不准
要可變：   箱子 x（標記可變）裝了 5  →  塞 6 進去 → ✅ 可以
```

為什麼這樣設計？因為**「會變的東西」是 bug 的溫床**。當一個值在程式各處被偷偷改來改去，你很難追蹤「它現在到底是多少、是誰改的」。Rust 預設不可變，等於逼你**只在真正需要時才開放修改**，讓程式更好推理、編譯器也能幫你擋住「不小心改到」的錯。

> 這呼應一個寫程式的好習慣：能不可變就不可變。在 TypeScript 你可能聽過 `const`；Rust 把這個精神變成預設值。

## 程式碼範例

### `let`：預設不可變

```rust
fn main() {
    let x = 5;
    println!("x 是 {}", x);
    x = 6;                  // ❌ 編譯錯誤！x 不可變
}
```

編譯器會明白地擋下來：

```
error[E0384]: cannot assign twice to immutable variable `x`
```

它甚至會貼心建議你「如果想改，加上 `mut`」。

### `mut`：我就是要可變

當你確實需要一個會變的變數，宣告時加上 `mut`（mutable，可變的）：

```rust
fn main() {
    let mut x = 5;
    println!("一開始 x 是 {}", x);
    x = 6;                  // ✅ 可以，因為宣告成 mut
    println!("現在 x 是 {}", x);
}
```

關鍵在於：**「可變」是你主動、明確要求的，而不是預設就送你。** 讀程式的人看到 `mut` 就知道「這個值之後會變」，這本身就是有用的資訊。

### `const`：真正的常數

`const` 是「永遠不變、編譯時就確定」的常數，習慣用**全大寫加底線**命名：

```rust
const MAX_RETRY_COUNT: u32 = 3;
```

`let`（不可變）和 `const` 的差別：`const` 更嚴格（值必須在編譯期就能算出來、一定要寫型別、可放在函式外當全域常數）。當你要表達「這是一個固定的設定值」，用 `const`，順便消滅程式裡的「魔術數字」。

> 「魔術數字」（直接寫 `3` 而不說明 3 是什麼）是常見的反模式 → [課外讀物 E-6-6：程式碼異味與反模式](../../../課外讀物/E-6-best-practices/E-6-6-anti-patterns.md)

### 遮蔽（shadowing）：同名再宣告

Rust 允許你用同一個名字「再 `let` 一次」，這叫**遮蔽**。它和 `mut` 不同——遮蔽是**建立一個新變數**，剛好沿用舊名字：

```rust
fn main() {
    let spaces = "   ";          // 字串
    let spaces = spaces.len();   // 遮蔽：現在 spaces 是數字(長度)
    println!("空白數量：{}", spaces);
}
```

遮蔽好用在「同一個概念、轉換成另一種型別」的時候——你不用硬想第二個變數名。注意這和 `mut` 的差別：`mut` 不能改變型別，遮蔽可以（因為它根本是新變數）。

## 小練習

1. 宣告一個不可變變數存你的年齡並印出。接著試著改它的值，觀察編譯器的錯誤訊息，再加上 `mut` 讓它通過。
2. 用 `const` 定義一個「圓周率」常數 `PI`（值 `3.14159`），算出半徑 10 的圓面積並印出。想想為什麼用 `const` 比直接在公式裡寫 `3.14159` 好。
3. 用「遮蔽」把一個字串 `"42"`（先別管怎麼轉，這題只要體會遮蔽）重新 `let` 成同名變數，賦一個數字值，印出來。觀察：同名是允許的嗎？
