# [rust-1-4] 函式與「表達式導向」：為什麼 Rust 的 if 可以當值用

> **本章目標**：學會定義與呼叫函式，並理解 Rust 一個很重要的特色——它是「表達式導向」語言，這讓函式回傳值寫起來特別簡潔。

## 你會學到

- 用 `fn` 定義函式、宣告參數與回傳型別
- 「敘述（statement）」與「表達式（expression）」的差別
- 為什麼函式最後一行「不加分號」就是回傳值
- 這個特性怎麼讓 `if` 也能當值用

## 概念說明

### 函式：把一段工作包起來、取個名字

函式就是「把一段做某件事的程式碼包起來、給它一個名字」，之後要用就喊名字。一個比喻：

```
定義一份「泡咖啡」的食譜（函式）：
    輸入：咖啡豆克數、水量      ← 參數
    做一連串步驟
    產出：一杯咖啡             ← 回傳值

之後想喝就喊「泡咖啡(20, 300)」，不用每次重寫步驟
```

> 一個函式最好只做一件事、名字能說明它在做什麼——這是寫好函式的根本原則 → [課外讀物 E-6-3：函式的設計](../../../課外讀物/E-6-best-practices/E-6-3-function-design.md)

### 表達式 vs 敘述：Rust 的關鍵特色

這是 Rust 和很多語言不同、值得花點時間懂的地方：

- **敘述（statement）**：做一件事，但**本身不產生值**。例如 `let x = 5;`。
- **表達式（expression）**：會**算出一個值**。例如 `5 + 3` 算出 `8`，`x + 1` 也算出一個值。

Rust 是**表達式導向**的語言——很多東西都會「算出一個值」，包括 `if`、區塊 `{}` 等。這帶來一個招牌寫法：**函式的最後一個表達式，如果不加分號，就是這個函式的回傳值。**

```
有分號 → 是「敘述」，不回傳值
沒分號 → 是「表達式」，它的值就是回傳值
```

剛開始你會覺得「少一個分號差這麼多？」對，這正是 Rust 風格。

## 程式碼範例

### 基本函式：參數與回傳型別

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b          // 注意：沒有分號 → 這就是回傳值
}

fn main() {
    let sum = add(3, 5);
    println!("3 + 5 = {}", sum);
}
```

逐項說明：

- `fn add(...)`：定義函式 `add`。
- `(a: i32, b: i32)`：兩個參數，**參數一定要寫型別**（函式邊界上 Rust 不幫你猜）。
- `-> i32`：箭頭後面是**回傳型別**。
- `a + b`（無分號）：函式的最後一個表達式，它的值就是回傳值。

如果你手癢加了分號寫成 `a + b;`，它就變成「敘述」、不回傳值，編譯器會抱怨「你說好要回傳 `i32`，怎麼沒給？」。你也可以用明確的 `return a + b;`，但 Rust 慣例是「最後一行不加分號」這種簡潔寫法。

### `if` 也是表達式，能當值用

因為 `if` 會「算出一個值」，你可以直接把它賦給變數：

```rust
fn main() {
    let score = 72;
    let level = if score >= 60 { "及格" } else { "不及格" };
    println!("結果：{}", level);
}
```

說明：整個 `if ... else ...` 算出一個字串，直接指派給 `level`。注意兩個分支要回傳**同一型別**（這裡都是字串）。在 TypeScript 你得用三元運算子 `score >= 60 ? "及格" : "不及格"`；Rust 直接用 `if` 就行，更好讀。

### 區塊 `{}` 也是表達式

連大括號區塊都能算出值（最後一行不加分號）：

```rust
fn main() {
    let y = {
        let a = 3;
        let b = 4;
        a * b          // 無分號 → 這個區塊算出 12
    };
    println!("y = {}", y);
}
```

這在你想「先做幾步計算，再把結果存起來」時很乾淨。

## 小練習

1. 寫一個函式 `square(n: i32) -> i32` 回傳 `n` 的平方（用「最後一行不加分號」的寫法），在 `main` 呼叫並印出 `square(7)`。
2. 寫一個函式 `is_even(n: i32) -> bool`，用 `if` 表達式回傳這個數是不是偶數（提示：`n % 2 == 0`）。
3. 把練習 1 的函式最後一行加上分號，編譯看看錯誤訊息，理解「分號讓它從表達式變敘述」造成的差別。
