# [rust-2-6] 可變借用與借用規則：為什麼「同時只能有一個可變借用」能防止一整類 bug

> **本章目標**：學會可變借用 `&mut`，並理解 Rust 的借用規則——這套規則正是 Rust 能在編譯期擋掉「資料競爭」、達成「無懼並行」的祕密武器。

## 你會學到

- 用 `&mut` 借來一個值並修改它
- 借用的兩條黃金規則
- 為什麼「同時只能有一個可變借用」聽起來綁手綁腳，其實在救你

## 概念說明

### 可變借用 `&mut`：借來而且能改

上一章的 `&` 是唯讀借用。想「借來並修改」，用 `&mut`（mutable reference，可變參考）：

```rust
let mut s = String::from("哈囉");
change(&mut s);          // 借出一個「可變參考」
```

但 Rust 對可變借用設下嚴格規定。這就是**借用規則**：

```
規則一：在任何一個時間點，對同一份資料，你只能有以下其中一種：
        ● 任意數量的「唯讀借用」(&)   —— 大家一起讀，沒問題
        ● 「只有一個」可變借用 (&mut) —— 要改，就只能你一個人

規則二：參考必須永遠有效（不能指向已經被清掉的資料）。
```

一句話濃縮規則一：**「要嘛多個讀者、要嘛一個寫者，兩者不能同時。」**

```mermaid

    OK1["✅ 多個 &（唯讀）<br/>讀者A 讀者B 讀者C 同時讀"]
    OK2["✅ 一個 &mut（可變）<br/>唯一的寫者"]
    BAD["❌ 一個 &mut + 其他任何借用<br/>有人在寫，又有人在讀/寫 → 禁止"]

```

### 為什麼這條規則在救你？資料競爭

這條規則乍看綁手綁腳，但它防的是一個惡名昭彰的 bug——**資料競爭（data race）**。

想像一份共用資料，**一邊有人正在改它、同時另一邊有人在讀（或也在改）**。讀的人可能讀到「改到一半」的殘缺資料；兩個一起改可能互相覆蓋。結果是隨機、難重現、超難除錯的 bug——這在多執行緒程式裡是頭號惡夢。

用生活比喻：一份 Google 文件，**一個人正在大改結構時，另一個人同時在讀**，讀的人會看到一團混亂的中間狀態。安全的做法是：要嘛大家都只讀，要嘛改的時候只有一個人動、其他人先等。

Rust 的借用規則「要嘛多讀、要嘛一寫」**精確對應**了這個安全準則。而且——**它是在編譯期強制檢查的**。所以 Rust 能在你還沒執行程式時，就擋掉資料競爭。這就是 Rust 著名的「**無懼並行（fearless concurrency）**」的根基（[rust-8-3] 會把它用在多執行緒上）。

## 程式碼範例

### 用 &mut 修改

```rust
fn main() {
    let mut s = String::from("哈囉");   // 注意 s 本身要是 mut
    change(&mut s);                     // 借出可變參考
    println!("{}", s);                  // 哈囉 world
}

fn change(text: &mut String) {          // 參數是 &mut String
    text.push_str(" world");            // 可以改了！
}
```

說明：三個地方要一致——`s` 要宣告成 `mut`、傳入時寫 `&mut s`、函式參數寫 `&mut String`。三者到齊，才能借來修改。

### 違反規則：同時兩個可變借用

```rust
fn main() {
    let mut s = String::from("哈囉");
    let r1 = &mut s;
    let r2 = &mut s;        // ❌ 編譯錯誤：不能同時有第二個可變借用
    println!("{} {}", r1, r2);
}
```

錯誤訊息：

```
error[E0499]: cannot borrow `s` as mutable more than once at a time
```

Rust 直接告訴你「同一時間不能借第二個可變參考」。在別的語言這種模式可能默默釀成資料競爭；Rust 連編譯都不給過。

### 違反規則：同時又讀又寫

```rust
fn main() {
    let mut s = String::from("哈囉");
    let r1 = &s;           // 唯讀借用
    let r2 = &mut s;       // ❌ 已經有唯讀借用了，不能再來個可變借用
    println!("{} {}", r1, r2);
}
```

只要「有人正在讀」，就不准「有人來改」——避免讀者讀到改到一半的資料。

### 規則沒那麼死：作用範圍結束就釋放

規則是「**同一時間**」。一個借用一旦用完（之後不再被使用），它就釋放了，你可以接著建立新的借用：

```rust
fn main() {
    let mut s = String::from("哈囉");
    {
        let r1 = &mut s;
        r1.push_str(" A");
    }   // r1 在這裡結束
    let r2 = &mut s;       // ✅ OK，因為 r1 已經不在了
    r2.push_str(" B");
    println!("{}", s);
}
```

說明：`r1` 用完（離開內層範圍）後，`r2` 就能合法地借了。所以借用規則限制的是「**同時**重疊」，不是「整段程式只能借一次」。實務上絕大多數程式自然就符合，沒你想的那麼綁手綁腳。

## 小練習

1. 寫一個函式 `fn append_excl(s: &mut String)`，在字串尾巴加上 `"!"`，在 `main` 用 `&mut` 呼叫它並印出結果。記得 `s` 要宣告成 `mut`。
2. 故意寫出「同時兩個 `&mut`」的程式，讀錯誤訊息 `cannot borrow ... as mutable more than once`。再用「加一對內層大括號」的方式讓兩個借用錯開、修好它。
