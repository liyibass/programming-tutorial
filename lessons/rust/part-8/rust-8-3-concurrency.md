# [rust-8-3] 並行（Concurrency）：執行緒，與「為什麼 Rust 的並行『無懼』」

> **本章目標**：學會用執行緒讓程式「同時做多件事」，並理解為什麼 Rust 能在編譯期就擋掉並行最可怕的 bug——這就是它著名的「無懼並行」。

## 你會學到

- 並行是什麼、執行緒（thread）是什麼
- 用 `thread::spawn` 開一個執行緒
- 並行最可怕的 bug：資料競爭
- 為什麼 Rust 能在編譯期擋掉它（呼應 Part 2）

## 概念說明

### 同時做很多事

到目前為止你的程式都是「一條線、從頭跑到尾」。**並行**是讓程式「**同時進行多件事**」——例如一邊下載檔案、一邊更新進度條。

做到這件事的基本單位是**執行緒（thread）**。比喻：

```
單執行緒：一個廚師，一道菜做完才做下一道。
多執行緒：好幾個廚師，同時各做一道菜 → 整桌菜更快上齊。
```

> 執行緒、行程、CPU 排程「怎麼假裝同時」的底層原理 → **cs 課程 Part 5：行程與執行緒、CPU 排程**

### 並行的惡夢：資料競爭

並行很powerful，但也帶來程式設計裡**最惡名昭彰的一類 bug**——**資料競爭（data race）**，[rust-2-6] 我們預告過：當多個執行緒**同時存取同一份資料、且至少一個在修改**，就會讀到壞掉的中間狀態、或互相覆蓋。結果是隨機出現、難以重現、超難除錯的災難。

在 C/C++、Java 裡，資料競爭**編譯時不會被發現**，要等到執行時（常常是上線後、在最糟的時機）才爆發。這是讓無數工程師掉頭髮的問題。

### Rust 的超能力：編譯期擋下資料競爭

這就是 Rust 最自豪的地方——**它的所有權與借用規則（Part 2），讓資料競爭在編譯期就無法通過**。

回憶 [rust-2-6] 的借用規則：「**要嘛多個唯讀借用、要嘛只有一個可變借用，不能同時。**」這條規則套到多執行緒上，恰好就是「防止資料競爭」的鐵則——不可能有「一個執行緒在寫、同時另一個在讀/寫」。Rust 的編譯器把這套規則延伸到執行緒之間，於是：

> **如果你的並行程式能編譯通過，它就不會有資料競爭。** 這就是「無懼並行（fearless concurrency）」——你可以放膽寫並行程式，因為編譯器幫你擋住了最可怕的那類 bug。

## 程式碼範例

### 開一個執行緒

```rust
use std::thread;

fn main() {
    // spawn 開一個新執行緒，跑裡面的閉包
    let handle = thread::spawn(|| {
        for i in 1..=3 {
            println!("子執行緒：{}", i);
        }
    });

    for i in 1..=3 {
        println!("主執行緒：{}", i);
    }

    handle.join().unwrap();     // 等子執行緒跑完才結束
}
```

說明：

- `thread::spawn(|| { ... })` 開一個新執行緒，執行那個**閉包**（[rust-6-5]）裡的程式。
- 主執行緒和子執行緒**同時進行**，所以印出的順序每次可能不同（這就是並行）。
- `handle.join()` 讓主執行緒「等」子執行緒跑完——不然 `main` 可能先結束，子執行緒還沒印完就被中止。

### 把資料移進執行緒：move

執行緒常需要用到外面的資料。因為執行緒可能活得比建立它的地方久，Rust 要求你用 `move` 把資料的**擁有權移進**閉包（呼應 [rust-6-5] 的閉包捕捉、[rust-2-3] 的移動）：

```rust
use std::thread;

fn main() {
    let data = vec![1, 2, 3];

    let handle = thread::spawn(move || {     // move：把 data 的擁有權移進來
        println!("子執行緒拿到：{:?}", data);
    });

    // println!("{:?}", data);   // ❌ data 已經被移進執行緒了，這裡不能用

    handle.join().unwrap();
}
```

說明：`move` 讓閉包**取得 `data` 的擁有權**。為什麼要這樣？因為子執行緒可能在 `main` 結束後還在跑，如果它只是「借用」`data`，那 `data` 被清掉後就成了懸空參考——Rust 用 `move` 強制「資料跟著執行緒走」，從根本上避免這問題。**所有權規則再次守護了你。**

### 那如果多個執行緒要共享同一份資料？

上面是「把資料給某一個執行緒」。但如果**多個執行緒要一起存取同一份資料**呢？這就不能只靠 `move`（擁有權只能給一個）。答案是下一節的 `Arc` + `Mutex`——專門為「執行緒間安全共享」設計的工具。

## 小練習

1. 用 `thread::spawn` 開三個執行緒，各自印出自己的編號（1、2、3）。觀察多次執行，順序是否每次都一樣。
2. 用 `move` 把一個 `String` 移進一個執行緒裡印出來，確認移進去後主執行緒不能再用它。
3. 思考題：用自己的話解釋「為什麼 [rust-2-6] 學的借用規則，剛好就能防止多執行緒的資料競爭？」
