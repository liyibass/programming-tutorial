# Infra 基礎建設課程大綱

> **核心理念**：把一台（或一群）機器「養好、跑穩、自動化」，是 infra 工程師的本事。
> 這門課假設你**完全沒學過 infra**，從「一台伺服器是怎麼運作的」開始，一步步帶你動手，最後你能獨立架設、維護、自動化一整套服務環境，成為一個稱職的 infra 工程師。

---

## 這門課的定位

| | 說明 |
|---|------|
| **適合對象** | 想搞懂「程式部署到伺服器之後發生什麼事」、想自己架服務、或想往 infra / DevOps 發展的人。**零基礎也能開始。** |
| **目標** | 學完能成為一個**稱職的 infra 工程師**：獨立管理 Linux 伺服器、架設服務、設好網路與安全、把維運自動化、做好監控與備份。 |
| **你需要準備** | **練習環境**——推薦用 Windows 的 **WSL**（Part 0 會帶你裝），日常 Linux 操作都在這裡練；再加一台 **雲端伺服器**（如 AWS EC2）做「對外伺服器」性質的實作。 |
| **這門課 vs AWS 課** | 這門課專注「**機器本身與自架（self-hosted）**」——OS、網路、服務、容器、自動化。AWS 課專注「**雲端託管服務**」。兩者互補：先懂 infra，再上雲會更踏實。 |

> 通用知識（終端機、Git、網路、安全）放在頂層 `課外讀物/`（E-1、E-3、E-8、E-10），這門課會在需要時引用，不重複造輪子。
> 標 🔧 的是**動手做**章節，標 🏆 的是**整合專案**。

> **練習環境怎麼選**：絕大多數章節（Part 1~2 的命令列、檔案、權限、行程、套件…）在 **WSL** 上練就完美，零差異。少數「對外伺服器」性質的實作（防火牆對外、公開 HTTPS、被別人 SSH 登入）WSL 做不到位，建議用**雲端伺服器**（你的 AWS 帳號開一台 EC2）練。Part 0 會把這條界線講清楚。

---

## 學習路徑總覽

```
Part 0 練習環境（WSL）→ Part 1 認識伺服器 → Part 2 Linux 系統管理
   → Part 3 網路與連線 → Part 4 跑起一個服務（Nginx + systemd + HTTPS）
   → Part 5 容器化（Docker）→ Part 6 自動化與設定管理（腳本 + Ansible）
   → Part 7 觀測（日誌 + 監控）→ Part 8 可靠性與安全（備份 + 加固）
   → Part 9 從一台到多台 + 總整理專案
```

> 整門課圍繞「**你親手操作的 Linux 環境**」展開：先用 WSL 打好日常練習環境，每學一個概念就動手套用，最後把所學整合成一台架構完整、可監控、可一鍵重建的伺服器。

## Part 0 — 練習環境：用 WSL 打造你的 Linux 練習場

> 在 Windows 上，不用裝雙系統、不用開虛擬機，就能擁有一個完整的 Linux 環境——這就是 WSL。這個 Part 帶你把它裝好、設定好，之後所有 Linux 練習都在這裡進行。

### 章節列表

- `infra-0-1` WSL 是什麼？在 Windows 裡擁有一個真正的 Linux（WSL vs 雙系統 vs 虛擬機）
- `infra-0-2` 🔧 動手做：安裝並設定 WSL，打通與這門課的銜接（含啟用 systemd、安裝 Docker、與 Windows 檔案互通）

## Part 1 — Infra 是什麼？認識你的伺服器

### 章節列表

- `infra-1-1` Infra 工程師在做什麼？從「機房管理員」到「基礎設施工程師」的演變
- `infra-1-2` 一台伺服器的一生：從通電開機到對外提供服務（硬體 → 作業系統 → 服務）
- `infra-1-3` 伺服器住在哪？實體機 / 虛擬機 / 雲主機 / 容器的差別與取捨
- `infra-1-4` 🔧 動手做：登入你的 Linux 伺服器，認識它（我是誰、我在哪、它正在跑什麼）

## Part 2 — Linux 系統管理基礎

### 章節列表

- `infra-2-1` 檔案系統與目錄結構：`/etc`、`/var`、`/usr`、`/home` 各放什麼
- `infra-2-2` 使用者、群組與權限：`chmod` / `chown` / `sudo` 的世界
- `infra-2-3` 行程與資源：用 `ps` / `top` / `htop` 看誰在吃 CPU 和記憶體
- `infra-2-4` 磁碟與掛載：`df` / `du` / 分割區 / 掛載點是什麼
- `infra-2-5` 套件管理：用 `apt` / `dnf` 安裝、更新、移除軟體
- `infra-2-6` 🔧 動手做：建立一個非 root 使用者、設好 `sudo`、改用 SSH 金鑰登入並關掉密碼登入（伺服器安全第一步）

---

## Part 3 — 網路與連線

### 章節列表

- `infra-3-1` IP / Port / DNS 複習：封包怎麼找到你的伺服器（延伸自課外讀物 E-3）
- `infra-3-2` SSH 深入：金鑰原理、`~/.ssh/config`、跳板機（bastion）、port forwarding
- `infra-3-3` 防火牆基礎：`ufw` / `iptables`——只開該開的門（22 / 80 / 443）
- `infra-3-4` 查線路的工具：`ping` / `traceroute` / `dig` / `ss` / `curl` 怎麼用來除錯
- `infra-3-5` 🔧 動手做：用 SSH config 管理多台機器 + 設定防火牆只放行必要的埠

---

## Part 4 — 跑起一個服務

### 章節列表

- `infra-4-1` systemd 是什麼？讓你的程式變成「開機自動啟動、掛掉自動重啟」的服務
- `infra-4-2` 🔧 動手做：把一個程式寫成 systemd service，並用 `journalctl` 看它的日誌
- `infra-4-3` Nginx 入門：反向代理、靜態檔案、設定檔結構（用「大樓櫃台」來理解）
- `infra-4-4` HTTPS 憑證：用 Let's Encrypt + certbot 免費幫網站上 HTTPS
- `infra-4-5` 🏆 動手做：在你的伺服器上，用 Nginx + systemd + HTTPS 完整跑起一個對外網站

---

## Part 5 — 容器化

### 章節列表

- `infra-5-1` 為什麼要 Docker？解決「在我電腦上明明可以跑」的世紀難題
- `infra-5-2` 四大概念：Image / Container / Volume / Network
- `infra-5-3` Dockerfile：把你的服務一層一層打包成 image
- `infra-5-4` Docker Compose：用一個 YAML 檔編排多個容器（app + 資料庫 + 快取）
- `infra-5-5` 🔧 動手做：把 Part 4 的網站容器化，用 `docker compose up` 一鍵啟動整套環境

---

## Part 6 — 自動化與設定管理

### 章節列表

- `infra-6-1` Shell scripting 實用篇：把重複的維運動作寫成可重跑的腳本
- `infra-6-2` Cron 與排程：定時備份、清日誌、跑維護任務
- `infra-6-3` 為什麼「手動設定」遲早會出事？設定漂移（configuration drift）與 IaC 思維
- `infra-6-4` Ansible 入門：用 playbook 一次設定多台機器，不用一台台 SSH 進去改
- `infra-6-5` 🔧 動手做：寫一個 Ansible playbook，自動裝好 Part 4 的整套環境（之後換機器一鍵重現）

---

## Part 7 — 觀測：知道伺服器發生什麼事

### 章節列表

- `infra-7-1` 日誌在哪？`journald`、`/var/log`、以及日誌輪替（`logrotate`）避免硬碟被塞爆
- `infra-7-2` 該看哪些指標？CPU / 記憶體 / 磁碟 / 網路的健康訊號
- `infra-7-3` Prometheus + Grafana 入門：把指標收集起來、畫成圖、設定告警
- `infra-7-4` 🔧 動手做：在伺服器上裝好 Prometheus + Grafana，監控你自己的服務與主機

---

## Part 8 — 可靠性與安全

### 章節列表

- `infra-8-1` 備份策略：3-2-1 原則——備份要能「還原」才算數
- `infra-8-2` 🔧 動手做：設定自動備份，並**實際演練一次還原**（沒演練過的備份等於沒有）
- `infra-8-3` 伺服器加固（hardening）：關閉沒用的服務、自動安全更新、用 fail2ban 擋暴力破解
- `infra-8-4` 災難復原：機器整台掛了怎麼快速重建（這就是為什麼前面要 IaC）

---

## Part 9 — 從一台到多台

### 章節列表

- `infra-9-1` 反向代理與負載平衡：用 Nginx 把流量分給多台後端
- `infra-9-2` 高可用概念：單點故障（SPOF）是什麼、冗餘怎麼做
- `infra-9-3` 什麼時候該從「自架」搬上雲？自架 vs 雲端託管的真實取捨（橋接到 AWS 課程）
- `infra-9-4` 🏆 總整理專案：用「Ansible（設定）+ Docker Compose（服務）+ Prometheus/Grafana（監控）+ 自動備份」，把你的伺服器打造成一套**可重建、可監控、可還原**的完整環境
