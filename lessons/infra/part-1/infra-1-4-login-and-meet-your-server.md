# [infra-1-4] 🔧 動手做：登入你的 Linux 伺服器，認識它

> **本章目標**：實際用 SSH 連進你的 Linux 伺服器，用三個問題「我是誰、我在哪、它在跑什麼」摸清楚這台機器的底細。

## 你會學到

- 怎麼用 SSH（Secure Shell）安全地登入遠端伺服器
- 三組「認識一台機器」的基本指令
- 怎麼看出這台機器的身分、規格、健康狀態與正在跑的服務
- 把前三章學到的概念，第一次真正套用到你自己的機器上

## 概念說明

### 先說：怎麼「進到」一台沒有螢幕的機器？

前面說過，伺服器通常**沒有螢幕和鍵盤**，它放在機房或雲端的某個角落。那我們要怎麼操作它？

答案是 **SSH（Secure Shell，安全外殼協定）**。它讓你從自己的電腦，透過網路「遠端登入」到伺服器，之後你在自己終端機裡打的每個指令，都會送到那台遠方的機器上執行。

用打電話來類比：

```
你的電腦（打電話的人）
   →  撥號（ssh 指令 + 伺服器位址）
   →  對方接起來、確認你是誰（驗證身分）
   →  接通！（你打的字都送到對方那台機器執行）
```

> SSH 是 infra 工程師每天用到的頭號工具。這章先學會「連進去」，Part 3 會再深入 SSH 的金鑰、設定檔等進階用法。

---

### 登入後，問機器三個問題

連進去之後，先別急著做事。一個老練的 infra 工程師，到一台陌生機器的第一件事，是先「認識它」——問三個問題：

```mermaid
graph LR
    A["1. 我是誰？<br/>用哪個帳號登入、權限多大"]
    B["2. 我在哪？<br/>這台機器叫什麼、是什麼系統、什麼規格"]
    C["3. 它在跑什麼？<br/>現在忙不忙、有哪些服務在跑"]

    A --> B --> C
```

這三個問題對應三組指令，下面一組一組帶你做。

## 程式碼範例

### 第一步：用 SSH 連進去

在你**自己電腦**的終端機輸入（把 `使用者名稱` 和 `伺服器IP位址` 換成你的）：

```bash
ssh 使用者名稱@伺服器IP位址
```

舉個實際例子，假設帳號是 `ubuntu`、伺服器 IP 是 `203.0.113.10`：

```bash
ssh ubuntu@203.0.113.10
```

第一次連線它會問你要不要信任這台機器，輸入 `yes`。接著依你的設定，輸入密碼或自動用金鑰登入。成功後，終端機開頭的提示字元會變成伺服器的，代表你**人已經在那台機器裡了**。

> 如果連不上，最常見的原因是 IP 打錯、帳號錯、或防火牆沒開 SSH 的門（第 22 號埠）。Part 3 會教你怎麼排查。

---

### 第二步：「我是誰？」

```bash
whoami
```

回傳你目前的登入帳號，例如：

```
ubuntu
```

這很重要——它決定你**能做什麼、不能做什麼**。如果回傳 `root`，代表你是「超級管理員」，權力最大也最危險（一個指令就能毀掉整台機器）。下一個 Part 我們會專門處理「不要直接用 root」這件事。

---

### 第三步：「我在哪？」

問機器的**名字**：

```bash
hostname
```

問它是**什麼作業系統、什麼核心版本**（呼應前一章的 Kernel 概念）：

```bash
uname -a
```

輸出大概像這樣：

```
Linux web-server-01 5.15.0-91-generic #101-Ubuntu SMP ... x86_64 GNU/Linux
```

`uname` 是 "unix name" 的縮寫，`-a` 代表 "all"（全部資訊）。你能讀到：它是 `Linux`、機器名叫 `web-server-01`、核心版本 `5.15.0`。

問它**規格**——記憶體多大、硬碟用了多少：

```bash
free -h
df -h
```

`free` 看記憶體（`-h` 是 "human-readable"，用 GB/MB 這種人看得懂的單位顯示），`df` 看硬碟空間（disk free）。infra 工程師超常用這兩個來判斷「機器是不是快撐不住了」。

`df -h` 輸出像這樣：

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/root        25G   8.2G   16G  35% /
```

這行在說：根目錄 `/` 總共 25G、用了 8.2G、還剩 16G、使用率 35%。當 `Use%` 逼近 100%，麻煩就要來了——這是 infra 最常見的事故之一。

---

### 第四步：「它在跑什麼？」

看機器**開機多久、現在忙不忙**（呼應第 1、2 章）：

```bash
uptime
```

看**目前有哪些服務正在跑**：

```bash
systemctl list-units --type=service --state=running
```

看**即時的資源使用狀況**（哪個程式在吃 CPU 和記憶體）：

```bash
top
```

`top` 會開啟一個會即時更新的畫面，列出最吃資源的程式。看完之後，按鍵盤的 `q`（quit）就能離開。

---

### 做完了，怎麼「掛電話」？

認識完機器，要離開 SSH 連線，輸入：

```bash
exit
```

你就會回到自己的電腦。記住：`exit` 只是斷開連線，**那台伺服器還是繼續開著、繼續服務**——這正是伺服器和個人電腦最大的不同。

## 小練習

### 練習 1：替你的伺服器做一張「身分證」

實際登入你的伺服器，跑過上面的指令，把答案填進這張表：

| 問題 | 指令 | 你的伺服器的答案 |
|------|------|----------------|
| 我用哪個帳號登入？ | `whoami` | |
| 機器叫什麼名字？ | `hostname` | |
| 什麼作業系統 / 核心版本？ | `uname -a` | |
| 記憶體多大？ | `free -h` | |
| 硬碟用了幾 %？ | `df -h` | |
| 開機多久了？ | `uptime` | |
| 是不是虛擬機？ | `systemd-detect-virt` | |

填完這張表，你就完成了 infra 工程師面對新機器的「第一次問診」。

---

### 練習 2：驗證上一章的猜測

第 3 章結尾你猜過自己的伺服器是哪一種（實體機 / VM / 雲主機）。現在用 `systemd-detect-virt` 驗證看看，跟你猜的一樣嗎？

---

### 練習 3：觀察「它在跑什麼」

跑 `systemctl list-units --type=service --state=running`，看看你的伺服器上有哪些服務在跑。挑一個你認得的（例如 `ssh`），想想：如果這個服務掛掉了，會發生什麼事？

> 提示：如果 `ssh` 服務掛了，你下次就連不進這台機器了——這也是為什麼 infra 工程師對「服務的可靠性」這麼在意，這正是後面 Part 4 和 SRE 課程的主題。
